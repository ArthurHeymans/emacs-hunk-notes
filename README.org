#+title: hunk-notes.el
#+author: hunk-notes contributors

~hunk-notes.el~ lets you add local, inline review comments to unified diff
buffers in Emacs, then export those comments as a structured prompt for an AI
coding agent.

The package is intentionally local and VC-agnostic at its core: comments attach
to changed lines in ordinary ~diff-mode~ buffers. Git, Jujutsu, VC, Magit, and
AI-agent integrations are thin optional adapters around that core workflow.

* Status

Early implementation. The current package supports the first useful vertical
slice:

- Review any unified diff buffer with ~hunk-notes-mode~.
- Add/edit/delete/resolve inline comments.
- Render comments as yellow, read-only blocks directly inside the diff buffer.
- Navigate and list comments.
- Persist comments locally as Emacs Lisp sexps.
- Generate/copy/save/show an AI-agent Markdown prompt with only commented hunks by default.
- Open review buffers from Git and Jujutsu diffs.
- Soft handoff adapters for copy/buffer/shell, ECA, agent-shell, gptel,
  gptel-agent, pi-coding-agent, and ai-code-interface.el.

* Installation

Clone this repository and add it to your ~load-path~:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/emacs-hunk-notes")
(require 'hunk-notes)

;; Optional entry point modules:
(require 'hunk-notes-git)
(require 'hunk-notes-jj)
(require 'hunk-notes-vc)
(require 'hunk-notes-magit)
#+end_src

With ~use-package~:

#+begin_src emacs-lisp
(use-package hunk-notes
  :load-path "/path/to/emacs-hunk-notes"
  :commands (hunk-notes-current-diff-buffer
             hunk-notes-git-review-working-copy
             hunk-notes-jj-review-working-copy)
  :init
  (require 'hunk-notes-git)
  (require 'hunk-notes-jj))
#+end_src

* Quick start

** Review an existing diff buffer

1. Open or create a unified diff buffer.
2. Run:

#+begin_example
M-x hunk-notes-current-diff-buffer
#+end_example

3. Move point to an added, removed, or context line.
4. Add or edit a comment:

#+begin_example
C-c , c
#+end_example

Use ~C-c , C~ instead for a multi-line block editor.  If a region is active,
the comment attaches to the selected line range, not just the current line.

5. Generate/copy a prompt:

#+begin_example
C-c , y
#+end_example

If you want review commands available in ordinary diff buffers automatically,
enable the opt-in hook:

#+begin_src emacs-lisp
(setq hunk-notes-auto-enable-in-diff-buffers t)
#+end_src

Auto-enabled buffers keep comments hidden and use overlay rendering by default,
so the diff initially looks like a regular diff.  Use ~C-c , t~ or click the
~Hunk-Notes[...]~ mode-line lighter to show/hide comments.

** Review Git changes

#+begin_example
M-x hunk-notes-git-review-working-copy
M-x hunk-notes-git-review-revision
M-x hunk-notes-git-review-range
#+end_example

** Review Jujutsu changes

#+begin_example
M-x hunk-notes-jj-review-working-copy
M-x hunk-notes-jj-review-revision
M-x hunk-notes-jj-review-range
#+end_example

* Key bindings

When ~hunk-notes-mode~ is enabled:

| Key       | Command                                    |
|-----------+--------------------------------------------|
| ~C-c , c~ | ~hunk-notes-comment-dwim~              |
| ~C-c , C~ | ~hunk-notes-comment-block-dwim~        |
| ~C-c , x~ | ~hunk-notes-remove-comment~            |
| ~C-c , n~ | ~hunk-notes-next-comment~              |
| ~C-c , p~ | ~hunk-notes-previous-comment~          |
| ~C-c , l~ | ~hunk-notes-list-comments~             |
| ~C-c , t~ | ~hunk-notes-toggle-comments~           |
| ~C-c , r~ | ~hunk-notes-toggle-resolved~           |
| ~C-c , g~ | ~hunk-notes-show-prompt~               |
| ~C-c , y~ | ~hunk-notes-copy-prompt~               |
| ~C-c , s~ | ~hunk-notes-send-prompt-to-agent~      |
| ~C-c , q~ | ~hunk-notes-leave-review~              |
| ~C-c , ?~ | ~hunk-notes-dispatch~                  |
| ~C-c , d~ | ~hunk-notes-dispatch~                  |

The dispatch command uses a Magit-style Transient menu when ~transient~ is
available, and falls back to a compact one-key prompt otherwise.

* Commands

** Core review commands

- ~hunk-notes-current-diff-buffer~ :: Enable review mode in the current diff buffer.
- ~hunk-notes-comment-dwim~ :: Add a new comment at point, or edit the existing comment at point in the minibuffer.
- ~hunk-notes-comment-block-dwim~ :: Add or edit a comment at point in a dedicated buffer; save with ~C-c C-c~, cancel with ~C-c C-k~.
- ~hunk-notes-delete-comment~ :: Delete the comment at point after confirmation.
- ~hunk-notes-remove-comment~ :: Remove the visible comment block at point without confirmation. Empty edits also remove the comment.
- ~hunk-notes-toggle-resolved~ :: Toggle resolved/unresolved state.
- ~hunk-notes-next-comment~ / ~hunk-notes-previous-comment~ :: Navigate comments.
- ~hunk-notes-list-comments~ :: Show a simple comment list buffer.
- ~hunk-notes-toggle-comments~ :: Hide/show inline comment rendering while keeping review mode active.
- ~hunk-notes-show-comments~ / ~hunk-notes-hide-comments~ :: Explicitly show or hide inline comment rendering.
- ~hunk-notes-leave-review~ :: Disable review mode and remove rendered comments, leaving the buffer as a regular diff.
- ~hunk-notes-dispatch~ :: Open the review action menu.  This is a Transient menu when available, with a one-key fallback.

** Prompt commands

- ~hunk-notes-generate-prompt~ :: Return the Markdown prompt as a string.
- ~hunk-notes-copy-prompt~ :: Copy the prompt to the kill ring.
- ~hunk-notes-show-prompt~ :: Display the prompt in ~*hunk-notes-prompt*~.
- ~hunk-notes-save-prompt~ :: Save the prompt to a Markdown file.

** Persistence/export commands

- ~hunk-notes-save-review~ :: Save current review comments locally.
- ~hunk-notes-load-review~ :: Load saved comments for the current review.
- ~hunk-notes-export-review-json~ :: Export review metadata/comments as JSON.
- ~hunk-notes-list-saved-reviews~ :: Open the local review store in Dired.

** VC entry points

- ~hunk-notes-git-review-working-copy~
- ~hunk-notes-git-review-revision~
- ~hunk-notes-git-review-range~
- ~hunk-notes-jj-review-working-copy~
- ~hunk-notes-jj-review-revision~
- ~hunk-notes-jj-review-range~
- ~hunk-notes-vc-review-working-copy~
- ~hunk-notes-vc-review-revision~
- ~hunk-notes-vc-review-range~

* Agent handoff

The safest default is to copy or show a prompt, not auto-submit it. Auto-submit
is opt-in because coding agents may edit files or run commands.

#+begin_src emacs-lisp
(setq hunk-notes-default-agent 'copy)
(setq hunk-notes-auto-submit-to-agent nil)
#+end_src

Available adapter symbols include:

- ~copy~
- ~buffer~
- ~shell~
- ~agent-shell~
- ~eca~
- ~gptel~
- ~gptel-agent~
- ~pi~
- ~ai-code~

Send to an adapter with:

#+begin_example
M-x hunk-notes-send-prompt-to-agent
#+end_example

For CLI agents, configure a shell command that receives the prompt on stdin:

#+begin_src emacs-lisp
(setq hunk-notes-agent-shell-command "my-agent-command")
(setq hunk-notes-default-agent 'shell)
#+end_src

* Customization

#+begin_src emacs-lisp
;; Where local review sexps are saved.
(setq hunk-notes-storage-directory
      (locate-user-emacs-file "hunk-notes/reviews/"))

;; Diff scope in generated prompts: only commented hunks by default.
(setq hunk-notes-prompt-diff-scope 'commented-hunks)

;; Other choices:
;; (setq hunk-notes-prompt-diff-scope 'full)
;; (setq hunk-notes-prompt-diff-scope 'none)

;; Opt in to review commands in diff-mode/Magit/VC/Jujutsu diff buffers.
(setq hunk-notes-auto-enable-in-diff-buffers t)

;; Auto-enabled buffers default to a normal-looking diff.
(setq hunk-notes-auto-enable-show-comments nil)
(setq hunk-notes-auto-enable-render-style 'overlay)

;; Explicit starts show comments by default; change this if desired.
(setq hunk-notes-show-comments-on-enable t)

;; Restrict offered handoff adapters.
(setq hunk-notes-agent-backends '(copy buffer shell gptel pi))
#+end_src

The mode-line lighter shows comment state, for example ~Hunk-Notes[3 hidden]~ or
~Hunk-Notes[3 resolved:1]~.  Mouse-1 on the lighter toggles inline comment
visibility.

* Data model

Comments store more than only a line number. Each comment records:

- repository root
- backend
- base/target revisions
- file
- side: ~old~ or ~new~
- line/range
- change type: ~added~, ~removed~, or ~context~
- hunk header
- nearby anchor context
- body
- resolved state
- timestamps

The nearby context is intended for future robust re-anchoring when diffs change.

* Development

Run byte compilation with Emacs:

#+begin_src sh
emacsclient --eval \
  '(let ((default-directory "/path/to/emacs-hunk-notes/"))
     (add-to-list '\''load-path default-directory)
     (mapc #'byte-compile-file
           '\''("hunk-notes-core.el"
              "hunk-notes-diff.el"
              "hunk-notes-comments.el"
              "hunk-notes-overlays.el"
              "hunk-notes-storage.el"
              "hunk-notes-prompt.el"
              "hunk-notes-agents.el"
              "hunk-notes.el"
              "hunk-notes-git.el"
              "hunk-notes-jj.el"
              "hunk-notes-vc.el"
              "hunk-notes-magit.el")))'
#+end_src

Run the included ERT tests:

#+begin_src sh
server=hunk-notes-test
emacs -Q --daemon=$server
emacsclient -s $server --eval \
  '(progn
     (load-file "/path/to/emacs-hunk-notes/tests/hunk-notes-tests.el")
     (let ((failures nil)
           (total 0))
       (mapatoms
        (lambda (sym)
          (when (and (ert-test-boundp sym)
                     (string-prefix-p "hunk-notes" (symbol-name sym)))
            (setq total (1+ total))
            (let ((result (ert-run-test (ert-get-test sym))))
              (unless (ert-test-passed-p result)
                (push (list sym result) failures))))))
       (if failures
           (error "ERT failures: %S" failures)
         (format "ok %d tests" total))))'
emacsclient -s $server --eval '(kill-emacs)'
#+end_src
